═══════════════════════════════════════════════════════════════════════════
WHAT IS MCCFR?
═══════════════════════════════════════════════════════════════════════════

Standard CFR Algorithm:
  ❌ Traverses entire game tree every iteration
  ❌ O(|Tree|) complexity → doesn't scale to large games
  ✓ Zero variance in regret estimates

Monte Carlo CFR (MCCFR):
  ✓ Samples ONE history per iteration (stochastic tree walk)
  ✓ O(Tree Depth) complexity → scales to real poker!
  ❌ Non-zero variance in regret estimates

This step: COMPARE TWO WAYS TO BALANCE THIS TRADEOFF

═══════════════════════════════════════════════════════════════════════════
TWO MCCFR SAMPLING STRATEGIES
═══════════════════════════════════════════════════════════════════════════

1. EXTERNAL SAMPLING MCCFR
   ─────────────────────────
   What it samples:  Chance node outcomes only
   Agent actions:    Deterministic (best response)
   Variance:         Lower
   Exploration:      Slower but stable
   
   When to use: Kuhn Poker, games with few chance nodes, preference for
                stability over speed

2. OUTCOME SAMPLING MCCFR
   ──────────────────────
   What it samples:  ALL nodes (chance + agent actions)
   Agent actions:    Stochastic (from current policy)
   Variance:         Higher but may explore faster
   Exploration:      Faster but noisier
   
   When to use: Large strategy spaces, games with many chance nodes,
                when fast initial convergence matters

TECHNICAL TERMS YOU'LL ENCOUNTER:

• Exploitability = How much an opponent can earn vs. Nash strategy
• Nash Equilibrium = Strategies where no player can improve by deviating
• Information Set = Group of states player can't distinguish
• Counterfactual Value = Utility IF this node is reached
• Regret Matching = Convert regrets → probability distribution
• Monte Carlo = Using random samples instead of full enumeration
